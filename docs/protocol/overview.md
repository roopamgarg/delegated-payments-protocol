# Protocol Overview

> **Status:** Placeholder — v0.1 specification in progress.

The Delegated Payments Protocol (DPP) defines how users delegate bounded payment authority to AI agents and how merchants verify that authority.

## Core Concepts (planned)

- **Agent identity** — Cryptographic identity for agents acting on behalf of users
- **Capability tokens** — Attenuating, scoped delegation artifacts
- **Payment intents** — Structured requests for financial actions
- **Verification flows** — How merchants and payment rails validate delegation
- **Approval escalation** — Human-in-the-loop when OTP or explicit consent is required
